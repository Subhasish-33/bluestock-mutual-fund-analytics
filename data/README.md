# Bluestock Mutual Fund Analytics - Data Directory

This directory is the core data storage location for the project, tracking both raw inputs and processed outputs, including the centralized SQLite database.

## Directory Structure

- **`raw/`**: Contains original, untouched CSVs. Data ingested here is immutable.
  - `nav_history.csv`
  - `mutual_fund_metadata.csv`
  - `transactions.csv`
- **`processed/`**: Contains cleaned intermediate data artifacts generated by pandas before they are loaded into the database or dashboard. Includes metric exports like `cagr_report.csv` and `fund_scorecard.csv`.
- **`db/`**: Contains the local SQLite data warehouse.
  - `bluestock_mf.db`: The master relational database containing cleaned historical NAVs, metadata, user transactions, and performance metrics.

## Note on Version Control
To preserve repository size limits, raw datasets and most processed CSVs are ignored via `.gitignore`. The `bluestock_mf.db` and select essential output reports are explicitly tracked.
